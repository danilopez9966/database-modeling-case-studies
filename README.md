# Database Modeling Case Studies

A collection of real-world database design projects, including **ER diagrams, relational schemas and SQL implementations**.

This repository focuses on the **design phase of databases**, covering how to structure data efficiently, define relationships and apply normalization principles.

---

## About This Repository

This project contains multiple case studies where different systems are analyzed and transformed into well-structured relational databases.

Each case includes:

- Problem description
- Entity and relationship analysis
- ER diagram
- Relational schema
- SQL implementation
- Design decisions and improvements

## Concepts Applied

- Entity-Relationship (ER) modeling
- Normalization (1NF, 2NF, 3NF)
- Primary and foreign keys
- Constraints and data integrity
- Relationship types (1:1, 1:N, N:M)
- Schema design best practices
